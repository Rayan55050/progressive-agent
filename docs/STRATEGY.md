# Growth Strategy

## Phase 1: GitHub Release

- Clean up codebase, remove personal config, write documentation
- Publish on GitHub with MIT license
- Create README with quick-start guide, architecture overview, and screenshots
- Write CONTRIBUTING.md with clear guidelines for new contributors
- Set up CI/CD (GitHub Actions for tests and linting)

## Phase 2: Community Building

- Share on Reddit (r/selfhosted, r/LocalLLaMA, r/artificial), Hacker News, and dev communities
- Create a Telegram group for users and contributors
- Write blog posts about architecture decisions and lessons learned
- Respond to issues quickly, merge PRs generously, give credit openly
- Build a small core team of regular contributors

## Phase 3: Plugin Ecosystem

- Define a stable plugin API for tools, skills, and monitors
- Create a plugin registry (GitHub-based initially, dedicated site later)
- Write plugin development guide with examples
- Host community showcases — best tools, most creative skills
- Enable one-command plugin installation

## Phase 4: Enterprise Version

- Offer a managed hosting option for teams (multi-user, shared knowledge base)
- Enterprise features: audit logs, role-based access, compliance controls
- Professional support tier
- Keep core open source — enterprise features are additions, not locks on existing functionality
- Revenue funds continued open-source development

## Guiding Principle

Grow by being genuinely useful, not by marketing. Every new user should be able to set up the agent in under 10 minutes and see immediate value. Word of mouth from happy users beats any growth hack.
