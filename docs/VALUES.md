# Values

## Open Source

The code is public. Every tool, every skill, every line of the agent's personality. If it has access to your data, you have access to its source. Contributions welcome, forks encouraged. We believe open development produces better, safer software — especially when that software has access to your personal life.

## Privacy First

All data stays on your machine. Memory is SQLite on your disk. Embeddings are computed via API but stored locally. No telemetry, no analytics, no cloud sync. Your conversations are yours alone. We will never add features that require sending personal data to our servers, because we don't have servers.

## Self-Healing

The agent should recover from failures without human intervention. Watchdog processes, automatic restarts, fallback LLM providers, graceful degradation. If something breaks at 3 AM, it fixes itself by 3:01 AM. A personal assistant that needs babysitting isn't an assistant — it's a liability.

## Simplicity

Simple code is reliable code. Markdown files over complex plugin frameworks. TOML configs over admin panels. Direct tool calls over abstraction layers. If a junior developer can't understand a module in 5 minutes, it's too complex. We delete code more often than we add it.

## Reliability

The agent must work every day, not just on demo day. That means retries with backoff, rate limit handling, multi-provider fallbacks, persistent state, and thorough error handling. Boring reliability beats exciting fragility. We'd rather ship one tool that works perfectly than ten that work sometimes.

## Pragmatism

We prefer free APIs where viable, but accept paid services for reliability. We use battle-tested libraries over custom implementations. We optimize for "works today" over "perfect someday." Every architectural decision is made with a simple question: does this make the agent more useful to its owner?

## Community

No one builds great software alone. We value contributions of all kinds — code, documentation, bug reports, feature ideas, and translations. We respond to issues quickly, review PRs generously, and give credit where it's due. The project grows because people care about it, and we never take that for granted.

These values aren't aspirational — they're the decisions we make every day in code, design, and community.
